### 1.4 Design Patterns

**Visitor pattern (4 variants):**

- `AbstractVisitor` / `AbstractVisitorWithContext<ContextT>` — 38 abstract `visitXxx()` methods
- `PassThroughVisitor` / `PassThroughVisitorWithContext<ContextT>` — empty default implementations

**Transformer pattern (4 variants):**

- `AbstractTransformer<T>` / `AbstractTransformerWithContext<ContextT, T>` — 38 abstract `transformXxx()` methods
- `TransformerWithDefault<T>` / `TransformerWithDefaultAndContext<ContextT, T>` — default value implementations

**Double dispatch**: `accept()`, `acceptWithContext()`, `transform()`, `transformWithContext()` on `Class`

**Composite/Iterator**: `descendOnce()` (one level), `descend()` (recursive) — both implemented as generators

**Type guards**: Every type has `asXxx(that: Class): Xxx | null` + `isXxx(that: Class): that is Xxx` pair

**Null/Optional**: Most properties are `T | null`; `overXxxOrEmpty()` helpers provide safe iteration

## 4. stringification.ts

**Size**: 879 lines | **Role**: Bidirectional enum ↔ string conversion

### Dependencies

- `./types` (enum types only)

### Pattern: 3-function set per enum

For each enum:

- `xxxFromString(text: string): Enum | null` — returns null on failure
- `xxxToString(value: Enum): string | null` — returns null on failure
- `mustXxxToString(value: Enum): string` — throws `Error` on failure

Internally uses two bidirectional Maps: `FROM_STRING: Map<string, Enum>` and `TO_STRING: Map<Enum, string>`.

### Target Enums — 12 total

`ModelType`, `ModellingKind`, `QualifierKind`, `AssetKind`, `AasSubmodelElements`, `EntityType`, `Direction`, `StateOfEvent`, `ReferenceTypes`, `KeyTypes`, `DataTypeDefXsd`, `DataTypeIec61360`

**Note**: `DataTypeDefXsd` string representations include XSD namespace prefixes (e.g., `xs:anyURI`, `xs:base64Binary`).

## 6. verification.ts

**Size**: 7,836 lines | **Role**: Validates AAS instances against specification constraints

### Dependencies

- `./common` (AasCommon)
- `./constants` (AasConstants)
- `./types` (all AAS classes, AbstractTransformerWithContext)

### Error Types

- `PropertySegment`, `IndexSegment`, `Path` — similar to jsonization but `instance: AasTypes.Class`
- `VerificationError` — message + path

### Export Categories

**Regex pattern matching (~35 functions)**: `matchesXxx(text: string): boolean`

- `matchesIdShort` — `^[a-zA-Z][a-zA-Z0-9_]*$`
- `matchesBcp47` — BCP 47 language tag
- `matchesMimeType` — RFC 7231 MIME type
- `matchesRfc2396` — RFC 2396 URI
- `matchesXsDate`, `matchesXsDateTime`, `matchesXsDateTimeUtc`
- `matchesXsDouble`, `matchesXsFloat`, `matchesXsDecimal`
- `matchesXsLong`, `matchesXsInt`, `matchesXsShort`, `matchesXsByte`
- `matchesXsUnsignedLong`, `matchesXsUnsignedInt`, `matchesXsUnsignedShort`, `matchesXsUnsignedByte`
- `matchesXsBoolean`, `matchesXsBase64Binary`, `matchesXsHexBinary`, etc.

**Value validity checks (~15 functions)**: `isXxx(value: string): boolean`

- Pattern check + semantic validation (date validity, numeric ranges, etc.)
- `isXsLong` — checks 64-bit range via string comparison (works around JS `Number` precision limits)
- `isLeapYear` — leap year detection including BCE years

**Business rule validation (~20 functions)**:

- `langStringsHaveUniqueLanguages`, `qualifierTypesAreUnique`, `extensionNamesAreUnique`
- `idShortsAreUnique`, `valueConsistentWithXsdType`
- `isModelReferenceTo`, `isModelReferenceToReferable`
- IEC 61360 data specification validation functions

**Per-type string constraint verification (~15 functions)**: `verifyXxx(that: string): IterableIterator<VerificationError>`

- Length limits + pattern checks
- `verifyIdentifier` (1–2000 chars), `verifyNameType` (1–128 chars), `verifyIdShortType` (1–128 chars)

**Main entry point**: `verify(that: AasTypes.Class, recurse: boolean): IterableIterator<VerificationError>`

### Key Patterns

**Verifier class**: extends `AbstractTransformerWithContext<boolean, IterableIterator<VerificationError>>`

- `context: boolean` = whether to recurse into child properties

**Generator pattern**: all verification functions implemented as `function*` for lazy, memory-efficient evaluation

**Regex caching**: composed via `constructMatchesXxx()` helpers, cached as module-level constants

**AAS constraint references**: error messages include official spec constraint numbers such as `AASd-005`, `AASd-020`, `AASd-077`, `AASd-118`, `AASd-130`

## 8. Rust Porting Considerations

### types.ts → src/types/

| TypeScript                        | Rust equivalent                     |
| --------------------------------- | ----------------------------------- |
| `abstract class Class`            | `enum` + `enum_dispatch` crate      |
| Interfaces (IReferable, etc.)     | Rust traits                         |
| `T \| null` properties            | `Option<T>`                         |
| `AbstractVisitor`                 | Visitor trait (via `enum_dispatch`) |
| `AbstractTransformer<T>`          | Transformer trait                   |
| `function*` (descend)             | `impl Iterator`                     |
| `asXxx()` / `isXxx()` type guards | `as_xxx()` / `is_xxx()` methods     |
| `overXxxOrEmpty()`                | `.iter()` on `Option`               |

### common.ts → src/common/

| TypeScript                   | Rust equivalent                  |
| ---------------------------- | -------------------------------- |
| `Either<V, E>`               | `Result<V, E>`                   |
| `base64Encode/Decode`        | `base64` crate                   |
| `every()`, `some()`, `map()` | Built-in Iterator adapters       |
| `range()`                    | `0..n` range syntax              |
| `at()` (negative index)      | `slice.get()` or `.iter().rev()` |

### constants.ts → src/constants/

| TypeScript | Rust equivalent                         |
| ---------- | --------------------------------------- |
| `Set<T>`   | `HashSet<T>` or compile-time `phf::Set` |

### stringification.ts → src/stringification/

| TypeScript                  | Rust equivalent                                     |
| --------------------------- | --------------------------------------------------- |
| `Map<string, Enum>`         | `HashMap<&str, Enum>` or `match` expression         |
| `xxxFromString() → null`    | `fn from_str() -> Option<Enum>`                     |
| `mustXxxToString() → throw` | `fn to_str() -> &'static str` (or `unreachable!()`) |

### jsonization.ts → src/jsonization/

| TypeScript                         | Rust equivalent                   |
| ---------------------------------- | --------------------------------- |
| `Either<T, DeserializationError>`  | `Result<T, DeserializationError>` |
| `SetterForXxx` + `SETTER_MAP`      | `serde` crate or manual parsing   |
| `Serializer` (extends Transformer) | `serde::Serialize` implementation |
| `JsonObject`                       | `serde_json::Value`               |

### verification.ts → src/verification/

| TypeScript                                       | Rust equivalent                           |
| ------------------------------------------------ | ----------------------------------------- |
| `function*` (generator)                          | `impl Iterator<Item = VerificationError>` |
| `RegExp` (cached)                                | `regex` crate + `once_cell` / `LazyLock`  |
| `AbstractTransformerWithContext<bool, Iterator>` | Visitor pattern with context              |
| `isXsLong` (string comparison for 64-bit)        | `i64::from_str()` directly                |

### Recommended Crates

| Crate                  | Purpose                                                          |
| ---------------------- | ---------------------------------------------------------------- |
| `thiserror`            | Error type definitions (DeserializationError, VerificationError) |
| `anyhow`               | Top-level error handling                                         |
| `serde` + `serde_json` | JSON serialization/deserialization                               |
| `regex` + `once_cell`  | Regex caching                                                    |
| `base64`               | Base64 encoding/decoding                                         |
| `enum_dispatch`        | Dynamic dispatch for Visitor/Transformer pattern                 |
