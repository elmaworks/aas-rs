# IMPORTANT

## Do

- After writing code, always run linting and formatting.
- All work must successfully build without errors.
- Avoid using abbreviations. However, abbreviations are allowed for index variables used in for-loops or while loops, and for variables in lambda functions.

## DO NOT

- Do not use `std::any`.
- Do not use `Box<dyn std::error::Error>`.
- Do not use stdout for logging. i.e. `println!`, `print!`, or `eprintln!`.
